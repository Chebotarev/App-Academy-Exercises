After working with SQL yesterday, it was time to connect the concepts of database queries to the Ruby curriculum. Today we did that by implementing an object-relational mapping between the row entries in a database and custom Ruby objects. This involved first creating a singelton class that handles queries by inheriting from the SQLite3 gem. We then wrote a class for every table that allowed us to instantiate objects which encapsulated the data and write methods to simulate the relationships between the objects. We could then manipulate the database itself by calling methods on those objects with DML statements.

These exercises helped solidify my understanding of the way Ruby on Rails interacts with databases. By the end of the day, it was obvious that the objects my partner and I wrote emulated a lot of the functionality of the framework we were exposed to in the Hartl Rails Tutorial.
